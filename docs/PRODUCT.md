# AgentDesk

## 目标用户

这个工具面向同时使用多个 Claude / Codex 账号的人。典型场景是：一个工作号、一个个人号、一个备用号，或者多个团队账号。用户真正需要的不是保存账号密码，而是清楚知道：

- 哪个本地槽位对应哪个账号
- 哪个槽位的登录态、缓存、会话是独立的
- 某个账号下有哪些本地会话
- 某个会话什么时候新建、什么时候最后活跃、标题是什么、在哪个项目目录
- 如何把旧会话的信息一键交给新会话或另一个 AI 工具继续理解

## 产品边界

本工具不保存账号密码，也不绕过官方登录。每个账号槽位只是一个本地数据容器，登录和鉴权仍由 Claude / Codex 官方 App 完成。

## 核心对象

### 账号槽位

账号槽位是一套独立的本地数据目录。它保存官方 App 自己产生的登录态、缓存和本地数据。

字段：

- 应用：Claude 或 Codex
- 名称：用户自定义，例如「Claude 工作号」
- 分组：把槽位归到「工作 / 个人 / 备用」等分组，侧栏按分组归拢
- 备注：自由文本，给账号加说明
- 数据目录：该槽位的本地 profile 路径
- 会话根目录：用于扫描会话的位置
- 官方 App 可执行文件：通常自动发现，非标准安装时可手动指定
- 最近打开时间

### 会话

会话来自本地文件扫描，不要求用户手动录入。

字段：

- 标题
- 新建时间
- 最后活跃时间
- 项目目录
- 来源：Claude Code、Claude 本地、Codex、Codex 归档
- 状态
- 线程 ID
- 会话文件路径
- 可复制的稳定会话标识

## 主要工作流

1. 用户选择左侧账号槽位。
2. 中间表格自动显示该槽位下的会话。
3. 用户搜索或按最后活跃时间找到目标会话。
4. 右侧查看会话详情。
5. 一键复制交接信息，粘贴给新 Claude / Codex 会话。

## 跨平台路径

macOS:

- Claude 默认数据：`~/Library/Application Support/Claude`
- Codex 默认数据：`~/Library/Application Support/Codex`
- Codex 会话：`~/.codex`

Windows:

- Claude / Codex 默认 UI 数据：自动选择传统 `%APPDATA%` 或 Store/MSIX 的 `LocalCache\Roaming`
- Codex 会话：`%USERPROFILE%/.codex`
- 独立槽位：`%USERPROFILE%/.agentdesk/profiles`

Windows 独立槽位不能放在 AppData，否则 MSIX 文件系统虚拟化会让官方 App、AgentDesk 和 Explorer 看到不同位置。旧槽位只复制迁移，不自动删除原目录。

## 界面原则

默认庭院视图使用约 3:1 的左右结构：左侧是完整场景和内嵌 Agent 工作台，右侧是可独立滚动的账号、额度、提醒、会话和详情信息轨。经典视图保留原三栏工具台，随时可切回。

右侧信息轨在最小窗口高度下会压缩账号操作，把新增/编辑/移除收进「管理」，保证会话列表仍有可见区域；内容过长时滚动信息轨，不压扁或拉伸庭院画面。

不使用大段解释卡片。说明文字只出现在状态栏、按钮标题和必要字段名里。

首次打开显示一次性欢迎引导（四步流程 + 关键概念），之后不再自动出现；左上角「?」可随时重新打开。工作区本身不常驻教学卡。

## 产品化要求

0.2 版必须具备：

- 用户能看清账号目录和会话根目录不是同一个概念
- 用户能手动修改账号目录和会话根目录
- 用户能诊断为什么会话读不到或 App 打不开
- Windows 能解释当前使用的 executable / MSIX 数据目录，并修复旧 AppData 槽位
- 空状态要区分「没有任何会话」和「搜索没有命中」
- 复制交接信息默认不包含完整聊天内容
- macOS 和 Windows 都能产出可运行包
