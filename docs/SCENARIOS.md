# 使用场景拆解

## 场景 1：多个账号不串号

用户有多个 Claude / Codex 账号，希望同时登录或快速切换，但不希望登录态互相覆盖。

解决方式：

- 每个账号槽位绑定独立本地 profile 目录
- 点击「打开账号」时用该目录启动官方 App
- 管理器不保存密码，登录仍发生在官方 App

## 场景 2：找回某个旧会话

用户记得某个任务做过，但不知道在哪个账号、哪个会话里。

解决方式：

- 左侧按账号槽位切换
- 中间表格按最后活跃时间排序
- 搜索标题、项目路径或线程 ID
- 右侧显示会话文件和项目目录

## 场景 3：把旧会话交给新会话继续

用户希望把一个 Claude/Codex 旧会话的信息交给新会话，让新会话理解上下文。

解决方式：

- 选中会话
- 点击「复制交接信息」
- 粘贴到新 Claude / Codex 会话

复制内容只包含元信息：应用、账号槽位、标题、新建时间、活跃时间、项目目录、会话文件和线程 ID。默认不复制完整聊天内容，避免误传隐私。

## 场景 4：Windows 使用

Windows 用户也需要同样的账号槽位和会话索引能力。

解决方式：

- 自动识别传统 `%APPDATA%/<App>` 与 Store/MSIX 的 `LocalCache/Roaming/<App>` 实际目录
- Codex 会话目录使用 `%USERPROFILE%/.codex`
- 新增槽位放在 `%USERPROFILE%/.agentdesk/profiles`，避开 AppData 虚拟化
- 同时识别 Store 执行别名、MSIX 当前包、传统安装与旧版 `app-*` 更新目录
- 会话文件失效时回退到可访问的上级目录，不让 Explorer 弹“位置不可用”
- 旧 AppData 独立槽位支持一键复制迁移，旧目录保留
- Windows 版本提供 portable `.exe`
