一个 glyph，一部宪法，两张皮
——统一之后，怎么把细节做到位

骨架统一、控制条照原型落地之后，剩下的差距不在「哪块放哪」，而在节奏与秩序：留白取值游离、字号层级过多、同一行控件基线不齐、数字不等宽。本评审给出一部设计宪法（令牌化的取值纪律）+ 各分区的前后数值规范，并用真渲染（真 index.html + renderer + 场景引擎 + 提案 CSS 层）展示修改前后。方案未动产品代码。

壱

品牌：全站只有一个 glyph（已落地）

Brand

品牌识别的第一纪律是「同一个形象出现在所有触点」。顶栏的像素脸 是产品里唯一每天被看见的品牌物，App 图标却是另一套渐变插画——两套语系互相稀释。已按 .brand-mark 的 29 单位几何精确重绘全套图标（本项已实施并提交）。

旧 · 渐变插画语系（与界面像素风脱节）
新 · 顶栏 brand-mark 同一几何（#dc6b35 / #4a2f20 / #fff2d8 + 硬投影）

触点清单：macOS Dock（icns 全档原生重绘，16px 仍锐利）· Windows（ico，PNG 载荷多档）· 开发态 Dock（main.js 同步）· 顶栏（原生源头）。小尺寸不做缩放采样而是逐档重绘——像素风图形被插值采样会糊边，这是像素品牌的常见死因。

弐

设计宪法：令牌化的取值纪律

Constitution

现状的问题不是「难看」，是「没有法」：留白出现 8/9/10/11/12/14/16 七种取值，字号 10.5～20 共九档，同一行里按钮 30px、chips 26px、徽章自然高。宪法把「允许的取值」收成短清单——一致性来自减少可选项，不来自逐处校对。

§1 间距：4px 基线网格

令牌值用途
--sp-28px行内元素间隙（按钮之间、chip 之间）
--sp-312px组件内边距（卡片、工具条、单元格横向）
--sp-416px区块左右呼吸（控制条 / 表区 / 状态栏统一左缘）
规则：任何 CSS 里的留白必须取自 --sp-*；9/11/13 这类游离值一律吸附到最近令牌。全部区块共享同一条 16px 左缘线——左缘不齐是「散」感的第一来源。

§2 字阶：五级封顶 + 数字等宽

令牌值角色
--fs-caption11px表头 / 状态栏 / 字段标签（信息的"注脚层"）
--fs-body12.5px表体 / 详情值（阅读主体）
--fs-ui13px按钮 / 区块标题（操作层）
--fs-title14px名牌 / 卡片名（身份层）
时间、日期、百分比一律 tabular-nums：等宽数字让「活跃/新建」两列真正成为可扫读的列，而不是长短抖动的碎片。字重只留 400/600 两档。

§3 控件三档高：一行之内基线对齐

档高成员
lg30px主操作（打开账号 / 复制交接信息）——每屏只允许一个主操作色
md28px次级按钮 / 搜索框 / 管理下拉
sm24px额度 chips / 微件
规则：同一条横排里的控件必须取自相邻档，垂直居中共线。「说不出哪里不对」的错位感，多数来自行内控件高度私自为政。

§4 边界所有权 · 动效 · 焦点

每条接缝只归一个元素画（上块的 border-bottom 即接缝，下块不再画 top）——现状已合规，写进宪法防回潮。动效只留一档 120ms cubic-bezier(0.2,0,0,1)，hover/选中反馈全站同节奏，prefers-reduced-motion 全部退光。键盘焦点环全站一致：porcelain 蓝 / yard 橙，2px 外扩——现状焦点态覆盖不全，这是可访问性欠账。

伍

落地路线与风险

Rollout

本方案零结构改动、零皮肤语言改动——全部是取值与令牌。拍板后一次提交即可收编：

① 把 proposal.css 的 :root 令牌并入 styles.css 顶部（porcelain 层）；② 各分区数值按 §3 表逐条替换原值（都是等价选择器，直接覆写原声明处，不留 override 层）；③ yard 皮肤内四条同档对齐并入 yard.css；④ node --test 里把新令牌写成断言（宪法机验化），浏览器 harness 双视图×双主题四张截图验收。

风险评估
布局回归低——不动 grid/结构，只动 padding/height/font；固定窗口下无断点交互
皮肤冲突低——porcelain/yard 分层覆写各自令牌，已在真渲染双皮肤验证
可读性元数据降灰一级需在暗色下复核对比度（提案链接里给了暗色四页）
